## Dicas 3 ### Instalação Verificar disco: `lsblk` Gravar imagem: `dd bs=4M if=imagem.iso of=/dev/sde && sync` ### Retirando senha de PDF 1. `pdftops -upw SENHA arq_com_senha.pdf arq_sem_senha.ps` 2. `ps2pdf arq_sem_senha.ps arq_sem_senha.pdf` ### Inicialização lenta no *AntiX* Depois que fiz a modificação abaixo, o computador passou a inicializar mais rápido. [fonte](https://qastack.com.br/ubuntu/893817/boot-very-slow-because-of-drm-kms-helper errors) 1. `sudo nano /etc/default/grub` Adição de `video=SVIDEO-1:d` de forma que ficasse assim: `GRUB_CMDLINE_LINUX_DEFAULT="quiet splash video=SVIDEO-1:d"` 2. `sudo update-grub` 3. `sudo reboot` ### Problema do microfone no *AntiX* Tive um problema no microfone usando o *AntiX*. Já tive o mesmo problema em outras distribuições. Quando eu ia gravar algo tanto no *Audacity* quanto no *SimpleScreenRecorder* o som gravado saia apenas em um dos fones de ouvido. Com esta configuração isso foi consertado. **Lembando que** tive que passei a usar o *PulseAudio* em vez do *Alsa*. ```ssh sudo nano /etc/pulse/default.pa ``` Adicionar no final do arquivo: ```ssh .ifexists module-echo-cancel.so load-module module-echo-cancel aec_method=webrtc source_name=echocancel sink_name=echocancel1 set-default-source echocancel set-default-sink echocancel1 .endif ``` Fonte: [LinuxUprising](https://www.linuxuprising.com/2020/09/how-to-enable-echo-noise-cancellation.html) Para iniciar o *PulseAudio* o comando é: ```ssh pulseaudio --start ``` Para reiniciar (ou matar) o *PulseAudio*: ```ssh pulseaudio -k ``` Uma dica para o *AntiX* é inicial por padrão o *PulseAudio*. Ir em `control-centre > session > configuration of user`. Abrirá um arquivo, onde deverá ser inserido `pulseaudio -D &`. Depois, clicar com o botão direito no icone de volume, ir em preferências, e em `Status Icon`, `External mixer` você escreve *pavucontrol* em vez de *alsamixer*. [Fonte da dica.](https://www.antixforum.com/forums/topic/how-to-set-pulseaudio-default/) ### Problema do som no Firefox Ao assistir um vídeo no YouTube, o som do sistema automaticamente aumentava para 100%. Isso ocorria tando ao mudar de vídeo, quanto ao avançar no vídeo. Para corrigir: `sudo vim /etc/pulse/daemon.conf` Alterar a linha `flat-volumes = no` ### GNUPG Criptografar aquivos. Instalação: `sudo apt-get install gnupg gnupg2` 1) Gera uma chave `gpg --gen-key` 2) Lista as chaves `gpg --list-secret-keys` 3) Exportar a chave secreta `gpg --armor --export-secret-keys your_email@domain.com > your_name.asc` 4) Diretório do gpg `./gnupg` 5) Integra com o terminal - Editar `vim ~/.bashrc` - Vá para a última linha. - Inserir ``` export GPG_TTY=`tty` ``` 6) Integra com o VIM You can set plugin defaults in ~/.vimrc: `let g:GPGPreferArmor=1` `let g:GPGDefaultRecipients=["yourname@yourdomain.com"]` 7) Apagar chave `gpg --delete-keys 634EE583B18A14A78E3F5D84A17BEF835D67ABC1` ### Diretório e arquivos 1) Converta o diretório em arquivo: `tar czf myfiles.tar.gz mydirectory/` 2) Voltar à ser diretório: `tar xzf myfiles.tar.gz` 3) Encriptar: `gpg -e -r USERNAME filename.tar.gz` 4) Resulta em filemane.tar.gz.pgp 5) Decriptar: `gpg -d -o filemane.tar.gz filename.tar.gz.gpg` ### CRIPTOGRAFIA SIMPLES NO VIM (menos seguro) Criptografar: basta utilizar o seguinte comando quando o arquivo estiver aberto. `:set key=suasenha` Descriptografar: basta utilizar o mesmo comando com senha vazia `:set key=` > Fontes (GPG + Vim): > >https://www.lotharschulz.info/2018/10/21/transparent-editing-of-gpg-encrypted-files-with-vim/ > >https://www.geeksforgeeks.org/encrypting-and-decrypting-the-files-using-gnupg-in-linux/ > >https://statistics.berkeley.edu/computing/encrypt >
